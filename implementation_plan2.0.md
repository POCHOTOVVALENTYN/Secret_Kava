# 📋 План: Переробка Адмін Меню на задачно-орієнтований інтерфейс

## Мета
Замінити поточний технічний редактор таблиць на зрозуміле меню з конкретними розділами:
- **💰 Ціни** — перегляд і редагування цін по кожному аркушу
- **📋 Реєстри** — перегляд бронювань за поточний місяць

---

## 🗂 Нова структура Адмін Меню

```
🔐 Адмін Панель
│
├── 💰 Ціни та тарифи
│   ├── 👤 Ціни консультацій   (Реєстр спеціалістів)
│   │   └── [Ім'я] → Онлайн: X | Офлайн: Y
│   │            → [✏️ Змінити онлайн] [✏️ Змінити офлайн]
│   │
│   ├── 🏢 Ціни кабінетів      (Реєстр Кабінетів)
│   │   └── [Назва] → Тариф: X | Вечірній: Y
│   │            → [✏️ Змінити тариф] [✏️ Змінити вечірній]
│   │
│   └── 🎭 Ціни заходів        (Реєстр Заходів (Афіши))
│       └── [Назва] → Ціна: X грн
│                → [✏️ Змінити ціну]
│
└── 📋 Реєстри (поточний місяць)
    ├── 💻 Консультації онлайн  (Бронювання до спеціаліста, фільтр: онлайн)
    ├── 🛋 Консультації офлайн  (Бронювання до спеціаліста, фільтр: офлайн)
    ├── 🏠 Бронювання кабінетів (Бронювання кабінету)
    ├── 🎭 Реєстр заходів       (Реєстр Заходів (Афіши), лише актуальні)
    └── 🎟 Реєстр Афіш          (Бронювання на заходи (Афіши))
```

---

## ⚠️ Важливе питання перед реалізацією

> [!IMPORTANT]
> В аркуші `Бронювання до спеціаліста` є колонки: ID | Клієнт | Телефон | Спеціаліст | Дата | Час | Статус оплати | Сума.
> **Колонки "Формат" (онлайн/офлайн) немає.**
>
> Щоб розділити "Консультації онлайн" та "Консультації офлайн" — **потрібно додати колонку "Формат"** до аркуша та записувати туди значення при бронюванні.
> Або: показувати **всі консультації** в одному реєстрі без поділу.
>
> **Рекомендація**: додати колонку `Формат` після колонки `Спеціаліст`. Я додам запис цього поля у хендлер бронювань консультацій автоматично.

## 📐 Деталі реалізації

### Сценарій редагування цін (однаковий для всіх трьох розділів)

```
[Список імен/назв кнопками]
      ↓ вибір
[Картка з поточними цінами + кнопки "Змінити X"]
      ↓ вибір поля
[Повідомлення: "Введіть нове значення для X (зараз: Y)"]
      ↓ текст
[✅ Збережено! Нова ціна: Z]
      ↓ автоматично
[Повернення до картки спеціаліста/кабінету/заходу]
```

Запис у Google Sheets через `values().update()` у форматі A1-нотації.

### Сценарій перегляду реєстрів

```
[Кнопка розділу] → Завантаження...
      ↓
[Список записів за поточний місяць, по 5 на сторінку]
Формат рядка: Клієнт | Телефон | Спеціаліст | Дата | Час | Статус
      ↓
[◀️ Попередня] [Сторінка X/Y] [▶️ Наступна]
[⬅️ Назад до реєстрів]
```

Фільтр по місяцю — порівняння колонки `Дата` з `datetime.now().month` та `datetime.now().year`.

### Компонент 2: Хендлер адмін панелі

#### [MODIFY] [admin.py (handler)](file:///Users/valentin/Secret_Cava/app/bot/handlers/admin.py)
Повна переробка структури. Нові секції:

**Головне меню:**
- `adm:prices` → підменю цін
- `adm:registries` → підменю реєстрів
- `menu:main` → вихід

**Ціни — підменю:**
- `adm:p:spec` → ціни консультацій (список спеціалістів)
- `adm:p:room` → ціни кабінетів (список кабінетів)
- `adm:p:event` → ціни заходів (список заходів)

**Ціни — вибір запису:**
- `adm:pi:spec:{row_idx}` → картка спеціаліста з цінами
- `adm:pi:room:{row_idx}` → картка кабінету з тарифами
- `adm:pi:event:{row_idx}` → картка заходу з ціною

**Ціни — вибір поля:**
- `adm:pf:online:{row_idx}` → редагувати Ціна_Онлайн
- `adm:pf:offline:{row_idx}` → редагувати Ціна_Офлайн
- `adm:pf:hourly:{row_idx}` → редагувати Тариф_годинний
- `adm:pf:evening:{row_idx}` → редагувати Вечірній_тариф
- `adm:pf:eprice:{row_idx}` → редагувати Ціна (захід)

**Реєстри — підменю:**
- `adm:r:consult_on` → консультації онлайн
- `adm:r:consult_off` → консультації офлайн
- `adm:r:rooms` → бронювання кабінетів
- `adm:r:events_reg` → реєстр заходів
- `adm:r:afisha` → реєстр афіш (хто записався)
